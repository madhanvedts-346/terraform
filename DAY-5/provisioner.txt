what is provisioner in terraform?
In Terraform, a provisioner is a mechanism to execute scripts or commands on a resource after it is created. Essentially, provisioners allow you to do "post-deployment" configuration, like installing software or copying files.

Provisioners are not recommended for general configuration management—Terraform’s main job is infrastructure as code, not software provisioning—but they can be useful for simple tasks.

Types of Provisioners:
1.local-exec
Runs commands on the machine where Terraform is executed.
Eg:

resource "aws_instance" "web" {
  ami           = "ami-12345678"
  instance_type = "t2.micro"

  provisioner "local-exec" {
    command = "echo ${self.public_ip} >> ips.txt"
  }
}

2.remote-exec
Runs commands inside the remote server through SSH or WinRM.
Eg:

resource "aws_instance" "web" {
  ami           = "ami-12345678"
  instance_type = "t2.micro"
  key_name      = "new-key"

  connection {
    type        = "ssh"
    user        = "ec2-user"
    private_key = file("~/.ssh/id_rsa")
    host        = self.public_ip
  }

  provisioner "remote-exec" {
    inline = [
      "sudo yum update -y",
      "sudo yum install httpd -y",
      "sudo systemctl start httpd"
    ]
  }
}

3.file
Copies files from local machine to remote server.
Eg:
resource "aws_instance" "web" {
  ami           = "ami-12345678"
  instance_type = "t2.micro"
  key_name      = "new-key"

  connection {
    type        = "ssh"
    user        = "ec2-user"
    private_key = file("~/.ssh/id_rsa")
    host        = self.public_ip
  }

  provisioner "file" {
    source      = "app.sh"
    destination = "/tmp/app.sh"
  }

  provisioner "remote-exec" {
    inline = [
      "chmod +x /tmp/app.sh",
      "/tmp/app.sh"
    ]
  }
}
